Create documentation and using whichever tool or combination of tools for UML diagrams for the payment/invoicing system below

Description:
Allow users to make payments and get an online receipts, the receipts can be shared with selected group of people the user can also decide to make the receipt public, meaning anyone with the link can view it. Additionally the receipt can be downloaded as PDF. 

Opon succesful payment the receipt is generated and emailed to the user, If the has only a mobile number linked to his profile then a link to view the receipt is sent to his mobile phone, The link shouldn't be easy to guess


Organization:
an organization is an business entity that can receive payment, The organization has the ability to generate receipts/invoice and send it to the user for payment. 
The receipt will have a pay button that user can use to make the payment

Alternatively if the user knows exactly what he's paying for then he can generate the receipt while making the payment 

Users
Users can login via MobilePhone or email

Types of users/roles
- Normal user making the payment
- Organization - Organization can have an Admin that can Add banking information in addition to all the other roles and a cashier that can view transactions and create invoices/receipts to bill the client
